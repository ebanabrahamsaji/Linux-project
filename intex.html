sort

Definition:The sort command is used to arrange the lines in a text file or input stream in a specific order, either alphabetically or numerically. By default, it sorts in ascending order, but options can modify this behavior. It is often used for organizing data files or logs. For instance, you can sort a list of names alphabetically or numbers numerically.

Purpose: To sort lines in a file or input.

Usage: sort file.txt

Example:

$ sort names.txt

awk

Definition: The awk command is a powerful text-processing tool designed for pattern scanning and data manipulation. It reads text line by line, searches for specific patterns, and performs actions like extracting fields, formatting data, or performing arithmetic. It is widely used in log analysis, data extraction, and reporting.

Purpose: To process and manipulate text data by applying patterns.

Usage: awk '{print $1}' file.txt

Example:

$ awk '{print $1}' data.txt

sed

Definition: The sed (stream editor) command is a versatile tool for making automated edits to text files or streams. It can search, replace, delete, or insert text based on patterns. It's commonly used in shell scripting for text preprocessing and transformation tasks.

Purpose: To transform text in files or streams efficiently.

Usage: sed 's/old/new/g' file.txt

Example:

$ sed 's/hello/hi/g' greetings.txt

cut

Definition: The cut command extracts sections of each line from a file or input stream, based on character positions, bytes, or delimiters. It is particularly useful for handling structured data, such as extracting specific columns from a CSV file.

Purpose: To extract specific columns or fields from a file.

Usage: cut -d',' -f2 file.csv

Example:

$ cut -d',' -f2 data.csv

tr

Definition: The tr command translates or deletes characters from the input. It's often used for tasks like converting text to uppercase or lowercase, replacing spaces with tabs, or removing unwanted characters from a stream.

Purpose: To transform characters in text by translating or deleting them.

Usage: echo 'abc' | tr 'a-z' 'A-Z'

Example:

$ echo 'hello' | tr 'a-z' 'A-Z'

echo

Definition:The echo command displays a line of text or the value of a variable in the terminal. It's frequently used in scripts to print messages, debug outputs, or pass data to other commands.

Purpose: To display text or variables in a terminal.

Usage: echo "Hello World"

Example:

$ echo "Hello World"

export

Definition:The export command sets and makes environment variables available to child processes. It is essential for passing configuration values like paths or user-specific settings in scripts or during session management.

Purpose: Make variables available to processes.

Usage: export VAR=value

Example:

$ export PATH=/usr/local/bin:$PATH

history

Definition:The history command shows a list of previously executed commands in the shell, helping users recall and reuse commands. It also allows re-execution of commands by their history number.

Purpose: Access previous commands quickly.

Usage: history

Example:

$ history | grep "git"

rsync

Definition: The rsync command is a fast and versatile utility for copying and synchronizing files and directories between two locations. It uses efficient data transfer methods, making it ideal for backups and remote file synchronization.

Purpose: Synchronize, transfer, and backup files.

Usage: rsync -av source/ destination/

Example:

$ rsync -av /source /backup

uniq

Definition: The uniq command filters out duplicate lines in a sorted file, leaving only unique lines. It is typically combined with the sort command for effective deduplication of data in text files.

Purpose: To find and display unique lines in a file or stream.

Usage: uniq file.txt

Example:

$ uniq sorted_file.txt
